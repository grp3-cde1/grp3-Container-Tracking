# Projektplan

## Vorgehensmodell

### Phase 1: Projektsetup
- GitHub-Repository erstellen
- Projektstruktur definieren
- Entwicklungsumgebung einrichten
- Grunddokumentation erstellen

### Phase 2: Einstieg mit Single-File-App
- erste Retrospektive-App in einer Datei umsetzen
- Python-Grundlagen anhand dieser App erklären
- CSV-Daten lesen und auswerten

### Phase 3: Strukturierung und OOP
- Code in Module aufteilen
- OOP-Konzepte einführen
- bessere Wiederverwendbarkeit schaffen

### Phase 4: Live-Daten
- REST-Schnittstelle verwenden
- MQTT-Daten abonnieren
- Live-Monitoring-App aufbauen
